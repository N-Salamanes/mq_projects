COMP348: Assignment 3 - Semantic Web

1. Introduction

This assignment consists of a basic task (1) and an advanced task (2) and is worth 15 marks in total.

  1. For the basic task, you have to write an XSL stylesheet that extracts weather forecast information (as specified below) from different XML documents and transforms this information into an XHTML document using the XSLT processor Saxon. The resulting XHTML document should be displayable in a Web browser (Google Chrome) with the help of a simple cascading sytlesheet (CSS).

  2. For the advanced task, you have to design (or reuse) an RDFa vocabulary that can be used to augment the information in the XSL stylesheet so that the generated XHTML document can finally be queried via SPARQL from a Python script that uses Python's rdflib library. Additionally, you have to describe the used RDFa vocabulary on a XHTML page.

2. XML Documents

We use the following seven XML documents published by the Bureau of Meteorology that contain weather information for a specific time period:

  • NSW and ACT Forecast (IDN10031.xml)
  • New South Wales (IDN11050.xml)
  • Sydney Metropolitan and Canberra (IDN11060.xml)
  • Victoria State Forecast (IDV10310.xml)
  • Victoria (IDV10753.xml)
  • Melbourne (IDV10751.xml)
  • Snowy Mountains District Forecast (IDN11032.xml)

You can download these seven XML documents as a Zip file (ass-3-xml-data.zip) from iLearn.

3. Basic Task (1)

Starting from the XML document IDN10031.xml, you have to write an XSL stylesheet (weather-1.xsl) that extracts the relevant information from the XML and transforms this information into an XHTML document (weather-1.html) with the help of a CSS stylesheet (weather.css) using the XSLT processor Saxon-HE (home edition).

Your XSL file should extract the above-mentioned information and fulfill in particular the following requirements. The XSL file should extract the text for:

  • the NSW and ACT Forecast from the XML document IDN10031.xml. It's up to you how you want to present this information on the web page;

  • Canberra and Sydney from the two XML documents IDN11050.xml and IDN11060.xml. Again, it's up to you how you want to present this information on the web page;

  • the Victoria State Forecast from the XML document IDV10310.xml;

  • Melbourne from the two XML documents IDV10751.xml and IDV10753.xml;

  • the Snowy Mountains District from the XML document IDN11032.xml. Make sure that the three locations are alphabetically ordered.

Note that the XML document IDN10031.xml is the main document in this task and the other XML documents have to be accessed via the XSL stylesheet (weather.xsl). You can access an XML document from an XSL stylesheet with the help of the XSLT document() function. The XHTML document that you generate with the help of the XSL stylesheet should contain a link to the CSS file (weather.css).

Since IDN10031.xml is the main document, you have to call the the XSLT processor Saxon in the following way:

  • C:\>java -jar saxon9he.jar IDN10031.xml weather-1.xsl > weather-1.html

This will generated the XHTML file weather-1.html as output.

4. Advanced Task (2)

In this advanced task, you will first augment the XSL stylesheet by adding RDFa markup to the XHTML content so that you can automatically answer at least four different types of SPARQL queries over the resulting XHTML document using a Python script. That means you have to keep the queries that you want to answer in mind when you design your RDFa markup.

Once you have augmented the XSL stylesheet (weather-2.xsl) with RDFa markup, you can generate a new XHTML document (weather-2.html):

  • C:\>java -jar saxon9he.jar IDN10031.xml weather-2.xsl > weather-2.html

Now you have to write a Python script (weather.py) that uses at least four different types of SPARQL queries of your choice to extract the relevant information from the XHTML document. The Python script should be able to fetch the XHTML document (weather-2.html) from an HTTP server. For this assignment, we will use a simple Python HTTP server that you can start up from the command line in the following way:

  • C:\>python -m http.server 8085

The Python script (weather.py) should now be able to access the XHTML document via

  • http://localhost:8085/weather-2.html

with the help of the Python's rdflib library:

     from rdflib import Graph

     if __name__ == '__main__':
         g = Graph()

         g.parse('http://localhost:8085/weather-2.html', format='rdfa')

     # Add SPARQL queries here.

The Python script should display the answers to the SPARQL queries on the standard output. Note that you have to provide an URI from the XHTML page (weather-2.html) to another XHTML document (weather-vocabulary.html) that describes the vocabulary used for the RDFa in natural language.

5. Assessment

The basic task (1) is worth 8 marks: 7 marks for the XSL stylesheet (weather-1.xsl) and 1 mark for the CSS stylesheet (weather.css). The advanced task (2) is worth another 7 marks: 2 marks for the augmented XSL stylesheet (weather-2.xsl), 4 marks for the Python script (weather.py) with the SPARQL queries, and 1 mark for the XHTML page (weather-vocabulary.html) that describes the RDFa vocabulary.

That means the whole assignment is worth 15 marks (= 15%) of the final marks for COMP348. You will be awarded marks according to the following criteria:

You have to submit a ZIP file (Ass3-StudentNumber-StudentName.zip) that contains the seven original XML documents, an XSL stylestheet (weather-1.xsl), a CSS stylesheet (weather.css), and the generated XHTML document (weather-1.html).

If you do the advanced task, then the ZIP file should additionally contain the augmented XSL stylesheet (weather-2.xsl), the generated XHTML document (weather-2.html), the Python script (weather.py), and the HTML page (weather-vocabulary.html) that describes the vocabulary.
